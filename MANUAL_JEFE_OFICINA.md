# Sistema de Tickets TIC

Guia ejecutiva y manual de uso para jefes de oficina.

---

## 1) Presentacion para direccion

### Que resuelve este sistema

Este sistema organiza y acelera la atencion de solicitudes de soporte tecnico en la oficina.  
En lugar de mensajes sueltos por chat o llamadas, todo queda en un solo lugar:

- quien reporto el problema
- que equipo presenta la falla
- quien del equipo tecnico lo esta atendiendo
- en que estado va
- cuando se cerro y con que solucion

### Beneficios para una jefatura

- **Orden y control:** cada caso queda registrado y no se pierde informacion.
- **Seguimiento claro:** puedes ver pendientes, en proceso y cerrados.
- **Trazabilidad:** sabes que tecnico atendio cada caso.
- **Mejor servicio:** el usuario recibe respuesta mas rapida y con evidencia.
- **Decisiones con datos:** hay estadisticas para ver carga de trabajo y tiempos.

### Por que se ve profesional

- Tiene portal para solicitudes y portal interno para tecnicos.
- Maneja permisos para evitar que cualquiera cambie informacion sensible.
- Guarda historial de atenciones.
- Permite generar ficha tecnica del servicio realizado.
- Incluye alertas y notificaciones para no dejar casos olvidados.

### Mensaje comercial corto (para presentar)

"Este sistema convierte el soporte tecnico en un proceso ordenado, medible y confiable.  
Reduce tiempos de respuesta, mejora la experiencia del usuario y permite a la jefatura tomar decisiones con informacion real."

## 2) Manual de uso (version jefatura)

## A. Como entra un usuario al sistema

1. Abrir la pagina del sistema.
2. En la pestana **Solicitar servicio**, llenar el formulario.
3. Enviar la solicitud.
4. El caso queda creado para el equipo tecnico.

---

## B. Como entra el personal tecnico

1. Ir a la pestana **Portal tecnico**.
2. Ingresar usuario/correo y contrasena.
3. Entrar al panel.

En el panel se ven tres vistas:

- **Pendientes:** casos nuevos por atender.
- **Completados:** casos ya cerrados.
- **Mis tareas:** trabajos asignados al tecnico.

---

## C. Flujo de atencion recomendado

1. Revisar la lista de **Pendientes**.
2. Tomar o asignar el ticket segun corresponda.
3. Cambiar a **En proceso** cuando se inicia atencion.
4. Registrar solucion aplicada.
5. Cerrar el ticket cuando el caso este resuelto.
6. Si aplica, descargar o consultar la ficha tecnica.

---

## D. Como hacer seguimiento desde jefatura

Revisar de forma diaria o semanal:

- Numero de tickets pendientes.
- Numero de tickets cerrados.
- Tecnicos con mayor carga de trabajo.
- Casos demorados o sin asignar.
- Tendencia general de atenciones.

Con esto puedes decidir:

- redistribucion de tareas
- prioridades por oficina
- necesidades de apoyo o recursos

---

## E. Buenas practicas de uso

- Registrar siempre datos completos del problema.
- Evitar cerrar tickets sin describir la solucion.
- Mantener actualizado el estado real del caso.
- Revisar diariamente los casos pendientes.
- Usar lenguaje claro en descripciones para facilitar auditoria.

## G. Preguntas frecuentes (FAQ rapido)

**1. Que pasa si un usuario no puede iniciar sesion?**  
Debe usar la opcion de reactivacion o comunicarse con administracion TIC.

**2. Se puede saber quien atendio un caso?**  
Si. Cada ticket guarda el responsable y el estado final.

**3. Se pueden ver estadisticas?**  
Si. El panel muestra indicadores de operacion y rendimiento.

**4. Que pasa si un ticket tarda mucho?**  
El sistema permite identificar demoras para priorizar atencion.
