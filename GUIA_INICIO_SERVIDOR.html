🚀 Sistema de Tickets

Guía Paso a Paso: Iniciar Servidores con PowerShell

1. Requisitos Previos

ℹ️ Antes de empezar, asegúrate de tener:
  • Windows con PowerShell disponible
  • Python 3.10+ instalado
  • Node.js y npm instalados
  • Entorno virtual (venv) configurado en el backend
  • Ambos equipos conectados a la misma red Wi-Fi
  • IP local del servidor: 192.168.20.25

2. Iniciar Servidor Backend (Django) - PowerShell

1
Abrir PowerShell
Abre PowerShell (búscalo en el menú Inicio de Windows o presiona Win + X y selecciona "Windows PowerShell").
2
Navegar a la carpeta del backend
En PowerShell, ejecuta:
cd c:\Users\sanch\Desktop\SistemaTickets\backend
3
Activar entorno virtual
En PowerShell, ejecuta:
& ..\venv\Scripts\Activate.ps1

Deberías ver (venv) al inicio de tu línea de comandos. Si da error de permisos, ejecuta primero:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Luego vuelve a ejecutar el Activate.ps1

4
Iniciar el servidor Django
En PowerShell, ejecuta:
python manage.py runserver 0.0.0.0:8080

Verás un mensaje como: Starting development server at http://127.0.0.1:8080/

⚠️ Importante

Mantén esta ventana de PowerShell abierta mientras uses el sistema. No la cierres.

3. Iniciar Servidor Frontend (React/Vite) - PowerShell

1
Abrir nueva ventana de PowerShell
Abre una segunda ventana de PowerShell (no cierres la primera que tiene el backend).

Atajo: Presiona Win + X → "Windows PowerShell (Admin)"

2
Navegar a la carpeta del frontend
En la nueva ventana de PowerShell, ejecuta:
cd c:\Users\sanch\Desktop\SistemaTickets\frontend
3
Iniciar el servidor Vite
En PowerShell, ejecuta:
npm run dev

Verás un mensaje como: VITE v5.0.0 ready in 500 ms ➜ Local: http://localhost:5173/

⚠️ Importante

Mantén AMBAS ventanas de PowerShell abiertas durante toda tu sesión de trabajo:

  • Primera ventana: Backend (Django)
  • Segunda ventana: Frontend (Vite)

7. Solución de Problemas - PowerShell

Error: "cannot be loaded because running scripts is disabled on this system"

Causa: PowerShell tiene restricciones de ejecución de scripts.

Solución: Ejecuta esto en PowerShell con permisos de administrador:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Presiona Y para confirmar. Luego intenta de nuevo:

& ..\venv\Scripts\Activate.ps1

El backend no inicia - Puerto ocupado

Error: "Address already in use" o "Permission denied"

Solución 1: Usa otro puerto:

python manage.py runserver 0.0.0.0:9000

Solución 2: Mata el proceso que ocupa el puerto 8080:

netstat -aon | findstr :8080

Luego ejecuta (reemplaza PID):

taskkill /PID [PID] /F

El frontend no inicia - npm no reconocido

Error: "npm is not recognized as an internal or external command"

Solución: Node.js/npm no está instalado. Descárgalo de:

https://nodejs.org/

Instálalo y reinicia PowerShell.

El frontend no carga desde la red

Error: "Cannot GET /" o página en blanco desde 192.168.20.25:5173

Solución: Verifica que el archivo vite.config.ts tenga la configuración de host:

server: { host: '0.0.0.0', port: 5173, }

Si no está, agrégalo y reinicia npm.

Error "DisallowedHost" al acceder desde IP

Error: "Invalid HTTP_HOST header: '192.168.20.25:8080'"

Solución: El archivo settings.py ya está configurado. Reinicia el backend:

python manage.py runserver 0.0.0.0:8080

Login rechazado con credenciales válidas

Error: "Credenciales inválidas"

Solución: Verifica que el usuario técnico existe. En PowerShell (carpeta backend):

& ..\venv\Scripts\Activate.ps1 python manage.py shell from django.contrib.auth.models import User User.objects.filter(email='tecnico@example.com').first() exit()

Si no existe el usuario, corre el script seed_data:

Get-Content seed_data.py | python manage.py shell

PowerShell no encuentra Python

Error: "python is not recognized"

Solución: Activa el entorno virtual primero:

& ..\venv\Scripts\Activate.ps1

El venv contiene la versión correcta de Python.

  5. Retorna JWT token de sesión
  6. Usuario accede al dashboard
